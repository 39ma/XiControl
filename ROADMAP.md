# Roadmap

Живой план. Не обязательства, а список идей по приоритету. Отметки: ✅ готово · 🔜 ближайшее · 💡 идея.

Актуально на **v0.5.3** (2026-07-10). История версий — в [CHANGELOG.md](CHANGELOG.md).

## ✅ Сделано

**Ядро (v0.1–v0.2)**
- Заряд «беречь ~80% / полный» через WMI + ChargeGuard (держится после сна/смены питания).
- Режимы производительности, трей + тёмное меню (следует теме на лету), автозапуск.
- OSD (питание, режимы, заряд), динамический значок трея по режиму.
- Авторские SVG-иконки (OSD цветные + трей монохром), рендер Svg.NET, app.ico.
- GitHub + CI-релизы по тегу (self-contained + framework-dependent exe).

**Клавиши (v0.3–v0.5)**
- Mi-кнопка: клик = цикл режимов, двойной клик = заряд 80/100, удержание = панель;
  при открытой панели жесты работают внутри неё. Раскладка и двойной клик настраиваются.
- «Настройки» (`0x1B`): заряд 80/100 (ремап на Параметры Windows), «Проекция» (`0x01`) → Win+P,
  AI (`0x23`) → Copilot или своя программа (`AiKeyProgram`), микрофон (`0x21`) → системный mute + OSD,
  подсветка (`0x05`) → OSD с уровнями (4 иконки), **Fn-Lock (`0x07`) → OSD с замком** (v0.5.3).
- Скрытый режим **Эко (`0x0A`)** — найден и подтверждён; Эко и Полную мощность можно скрыть.
- Панель: 5 режимов, анимации иконок (алгоритмические, 0% CPU в простое), Esc-хоткей.
- Подменю «Настройки» в меню трея: язык (RU/EN/ZH), автозапуск, галочки всех опций.
- README со скриншотами, LICENSE (GPLv3), CHANGELOG, документация протокола (две таблицы MIFS).

## 🔜 Ближайшее
- OSD «слабый/неправильный зарядник» (`0x01`/value 2) — событие уже ловим, но игнорируем.

## 💡 Идеи
- Настройки таймингов в config.json: порог удержания Mi, окно двойного клика, длительность OSD.
- Определение порога заряда по SKU (если тестеры подтвердят другие пороги — сейчас текст «80%» зашит).
- Камера/приватность (`0xA0` из карты MIControl) — на TM2424 событие не замечено, ждём тестеров.
- Переключатель частоты экрана / тачпада в меню (чистый Win32, прошивка не нужна).
- Экран «О системе» (модель, BIOS, серийник).

## ❓ Открытые вопросы
- Fn отдельным кодом не приходит → комбинации с Fn недоступны (обходим жестами Mi).
- Порог заряда на других моделях (70? 80?) — ждём отчётов тестеров.

## ✔ Закрытые вопросы
- WMI `0x0C` — рудиментный слот TPLock игровой таблицы MIFS: флаг хранится, к железу
  на TM2424 не подключён. Не используем. Детали: [docs/01-wmi-protocol.md](docs/01-wmi-protocol.md).
- Управление кулером сверх режимов — на TM2424 отсутствует (полный свип `0x01..0x30`:
  `0x14/0x15` макс. кулер и `0x17` CPU power — `0xE0`).

## 🚫 Не берём
- ICC-профили, splash-скины, служба + named pipe + protobuf (архитектура MIControl).
- GPU-режим, обороты/температуры — WMI на нашей модели не отдаёт.
- WinRing0 / прямой доступ к EC — цель проекта в том, чтобы обойтись без них.
- Lottie-анимации — заменены алгоритмическими (+10 МБ и второй рендер-стек не окупаются).
