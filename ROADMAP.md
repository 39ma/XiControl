# Roadmap

Живой план. Не обязательства, а список идей по приоритету. Отметки: ✅ готово · 🔜 ближайшее · 💡 идея.

## ✅ Сделано (v0.1)
- Заряд «беречь 80% / полный» через WMI + ChargeGuard (держится после сна/смены питания).
- Режимы производительности (Тихий/Авто/Турбо/Полная).
- Трей + тёмное меню, RU/EN, автозапуск (Планировщик, без UAC, на батарее).
- OSD (питание, смена режима/заряда) с тематическими иконками.
- Динамический значок трея по режиму.
- Mi-кнопка: короткое = цикл режимов + OSD, удержание = панель.

## 🔜 Ближайшее (Tier 1 — высокая ценность, инфраструктура готова)
Основано на разборе MIControl — см. [docs/06-more-from-micontrol.md](docs/06-more-from-micontrol.md).

- ✅ **Карта кодов клавиатуры** снята — см. [docs/07-keymap.md](docs/07-keymap.md).
  Мик `0x21` (инверт., не работает), проекция `0x01`, настройки `0x1B`, подсветка `0x05`, AI `0x23/24`, Mi `0x25/26`.
- **Оживить «мёртвые» клавиши** (шлют событие, но без софта ничего не делают):
  - «Настройки» (`0x1B`) → открыть Параметры Windows.
  - «Проекция» (`0x01`) → Win+P.
  - Скриншот (если есть) → «Ножницы».
  - Клавиша частоты экрана → цикл Гц.
  - Клавиша тачпада → вкл/выкл.
- **Микрофон** (`0x21`) → синхронизация системного mute + OSD.

## 💡 Позже (Tier 2 — обратная связь OSD)
- OSD для подсветки клавиатуры (`0x05`), Fn-Lock (`0x07`), камеры (`0xA0`).
- Предупреждение «слабый/неправильный зарядник» (`0x01`/arg 2).

## 💡 Опционально (Tier 3 — обычные функции Windows)
- Переключатель частоты обновления экрана в меню/панели.
- Тачпад вкл/выкл в меню.
- Экран «О системе» (модель, BIOS, серийник).

## 💡 Оформление и релиз
- Свой базовый значок (пользовательский `.ico`/`.png` или доработанный глиф).
- Публикация одним `.exe` (`dotnet publish -r win-x64 --self-contained -p:PublishSingleFile=true`).
- README проекта, скриншоты, лицензия (GPLv3).
- Настройки: порог «долгого» нажатия Mi, интервал OSD, вкл/выкл OSD — вынести в конфиг/меню.

## ❓ Открытые вопросы
- Fn отдельным кодом не приходит → комбинации с Fn недоступны (обходим жестами Mi).

## ✔ Закрытые вопросы
- WMI `0x0C` — исследовано (2026-07-10): записываемый флаг без наблюдаемого эффекта,
  гипотеза «тачпад» из Linux-драйвера не подтвердилась. Не используем.
  Детали: docs/01-wmi-protocol.md.

## 🚫 Не берём
- ICC-профили, splash-скины, служба + named pipe + protobuf (архитектура MIControl).
- GPU-режим, вентиляторы, температура — WMI на нашей модели не отдаёт (`0xE0`);
  полный свип `0x01..0x30` подтвердил: `0x14/0x15` (макс. кулер) и `0x17` (CPU power)
  из Linux-драйвера на TM2424 отсутствуют.
- WinRing0 / прямой доступ к EC — цель проекта в том, чтобы обойтись без них.
