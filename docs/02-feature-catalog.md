# Каталог функций — что можно положить в приложение

Легенда источника:
- 🟢 **WMI-native** — уникальная ценность, идёт через MIFS-прошивку.
- 🔵 **HW-event** — приходит WMI-событием (`HID_EVENT20`), нужен резидентный слушатель.
- ⚪ **Win32-generic** — стандартный Windows, к прошивке не относится, «бесплатно».

Оценка: **Core** — ядро продукта; **Nice** — приятное дополнение; **Later** — потом/по желанию; **Skip** — не тащить.

---

## Ядро (обязательно)

| Фича | Тип | cmd | Оценка | Комментарий |
|------|-----|-----|--------|-------------|
| Режим производительности | 🟢 | `0x08` | **Core** | Тихий / Турбо / Full-speed / Auto(Smart). Full-speed только на DC-БП. Auto только на `TM24*`. |
| Защита заряда 80/100% | 🟢 | `0x10` | **Core** | Бинарно. Порог зашит в прошивку: ≈80% на TM2424, на других моделях может отличаться (у MIControl — 70). Перед вкл — выкл (сброс EC). |
| Индикация в трее + меню | — | — | **Core** | Текущий режим/лимит, быстрое переключение. |
| OSD при переключении | — | — | **Core** | Как в CoreCharge — красиво и понятно (можно взять идею, не код). |
| Автозапуск при логине | — | — | **Core** | Task Scheduler, `/rl highest` (нужно для WMI Set — проверить). |

## Приятные дополнения

| Фича | Тип | cmd | Оценка | Комментарий |
|------|-----|-----|--------|-------------|
| Телеметрия: обороты вентиляторов | 🟢 | `0x0D` | **Nice** | CPU/GPU/SYS RPM. Хорошо смотрится в меню/тултипе трея. Только чтение. |
| Телеметрия: температура CPU | 🟢 | `0x16` | **Nice** | °C. Проверить код на железе (коллизия 0x16). |
| Тип БП (USB-C / DC) | 🟢 | `0x13` | **Nice** | Подсказать, почему Full-speed недоступен на USB-C. |
| Яркость подсветки клавиатуры | 🟢 | `0x12` | **Nice** | 0–10 + Auto(128). Слайдер в меню. Из Linux-драйвера — проверить. |
| Реакция на Fn-клавиши (OSD) | 🔵 | event | **Nice** | Показывать OSD когда режим/микрофон меняют физической клавишей. Требует слушателя событий. |
| Mi / AI кнопка → своё действие | 🔵 | event | **Nice** | Перехват кнопки, запуск приложения/команды (реестр). Фишка «как у больших». |
| Синхронизация mic-mute со звуком Windows | 🔵/⚪ | `0x0A`+audio | **Nice** | Аппаратный mute + системный mute вместе. |

## Позже / по желанию

| Фича | Тип | Оценка | Комментарий |
|------|-----|--------|-------------|
| Режим GPU (Hybrid/Discrete/UMA) | 🟢 `0x09` | **Later** | Мощно, но рискованно: обычно требует перелогина/перезагрузки, легко «уронить» вывод. Только после тестов. |
| Переключение частоты обновления | ⚪ | **Later** | Полезно на 120/165 Гц панелях, но это чистый `ChangeDisplaySettings`. |
| Вкл/выкл тачпада | ⚪ | **Later** | Обычный Windows-тумблер. |
| ICC цветовые профили | ⚪ | **Skip** | Нишевое, не про эту утилиту. |
| Произвольный % заряда (1–100) | ❌ EC | **Skip** | Осознанно отказались: требует WinRing0. Через WMI невозможно. |

## Мэппинг режимов (предложение для 3 кнопок в трее)

Чтобы UI был простым — три режима, как привыкли пользователи:

| UI | WMI arg (cmd 0x08) | Прим. |
|----|--------------------|-------|
| 🌿 Эко / Тихий | `0x02` (Quiet) | |
| 🧠 Умный / Авто | `0x09` (Auto) на `TM24*`, иначе `0x01` (Balance) | fallback для не-2025 |
| ⚡ Турбо | `0x03` (Turbo); опц. `0x04` Full-speed если на DC | |

## Итоговый MVP (рекомендация)

**v0.1:** трей + меню + OSD; режим производительности (0x08) и защита заряда (0x10); автозапуск; RU/EN/ZH.
**v0.2:** телеметрия (вентиляторы 0x0D, темп 0x16, тип БП 0x13) в тултип/меню; яркость клавиатуры (0x12).
**v0.3:** слушатель событий → OSD на Fn-клавиши + перехват Mi/AI-кнопки.

Всё в v0.1 делается **без службы** (см. `03-architecture.md`). Служба может понадобиться на v0.3 (события).
