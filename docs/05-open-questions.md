# Открытые вопросы — проверить на реальной машине ДО написания кода

Всё это выполняется в PowerShell от администратора на целевом ноутбуке.
Пробы с `operation=0xFA` (GET) — **только читают**, безопасны. Пробы с `0xFB` (SET) — меняют состояние, осторожно.

---

## 0. Хелпер для вызова метода

Сначала посмотреть сигнатуру метода:
```powershell
(Get-CimClass -Namespace root/wmi -ClassName MiCommonInterface).CimClassMethods
(Get-CimClass -Namespace root/wmi -ClassName MiCommonInterface).CimClassMethods['MiInterface'].Parameters
```
Это покажет точные имена параметров (`InData`/`OutData`) и их типы — нужно, чтобы правильно собрать вызов.

---

## Вопрос 1. Права: нужен ли SYSTEM для SET? 🔑

Определяет, можно ли обойтись без службы.

- [x] Установлено: **даже GET требует администратора** (без elevation `Get-CimInstance` → «нет доступа к CIM»). Значит приложению нужен `requireAdministrator`.
- [x] SET из обычного admin-процесса **работает** (включение защиты 0x02→1 прошло). → **Служба ради привилегий НЕ нужна.** Вопрос закрыт.
- [ ] Проверить: команда режима производительности реально сработала (вентиляторы/поведение) или вернула отказ.
- **Если admin достаточно** → служба не нужна (план А).
- **Если нужен SYSTEM** → минимальная служба для Set (план Б).

## Вопрос 2. Заряд: точный порог и «слетание» 🔑

- [x] Статус читается: `GET 0x10 sub=0x02` → `OUT[6]`.
- [x] ✅ **WMI-заряд подтверждён:** `0x02`=1 → стоп ≈80%, `0x02`=0 → заряд до 100% (доказано переключением, повторяемо). Держит WMI, не Windows.
- [x] Порог **фиксирован ≈80%** (не 70). Произвольный % недоступен. Модель для приложения = **бинарно 80/100**.
- [x] EC сбрасывает защиту при смене питания/сне → **реализован ChargeGuard** (`SystemEvents.PowerModeChanged`, переарм `0x02` с дебаунсом). ✅ Проверено на железе — держит надёжно.

## Вопрос 3. События: ловятся ли из user-сессии? 🔑

- [ ] Подписаться на `HID_EVENT20` и нажать Fn-клавишу (напр. Fn+F4 mic):
```powershell
Register-CimIndicationEvent -Namespace root/wmi -Query "SELECT * FROM HID_EVENT20" -SourceIdentifier mifs
# нажать Fn-клавишу, затем:
Get-Event -SourceIdentifier mifs | ForEach-Object { $_.SourceEventArgs.NewEvent.EventDetail }
```
- [x] ✅ **События ловятся из сессии.** Формат `[0]=01 тип, [1]=код, [2]=значение`. Коды: mic `0x21`, подсветка `0x05`, + `0x01/0x1B/0x23/0x24/0x25/0x26` (ярлыки уточнить по одной клавише). → **план А, служба не нужна.**

## 🏁 ИТОГ 05 — всё ключевое закрыто
- Права: admin хватает, **служба не нужна**.
- Заряд: **80/100 через WMI** (`0x10` sub `0x02`), подтверждён вживую.
- Режимы: **Quiet/Turbo/Full-speed/Auto** (`0x08`), Balance нет.
- События: **ловятся из сессии** (`HID_EVENT20`).
- Телеметрия: отсутствует на TM2424.
- Архитектура: **одно трей-приложение** (C#/.NET 8/WinForms, admin, автозапуск).
- Остаточное (не блокирует старт кода): ярлыки клавиш; сохранение заряда после сна (ChargeGuard); что такое `0x0C`.

## Вопрос 4. Режимы производительности на этой модели

- [x] Модель: SKU = `TM2424-77079` → префикс **`TM24`** ✅ → режим **Auto (0x09) доступен**.
- [x] Текущий режим читается: GET `0x08` → `OUT[4]` (был Auto 0x09).
- [x] SET работает: приняты **Quiet(0x02) / Turbo(0x03) / Full-speed(0x04) / Auto(0x09)**; Balance(0x01) отклонён. → 4 режима для UI.
- [ ] (опц.) Full-speed на USB-C vs DC — поведение (не критично для реализации).

## Вопрос 5. Телеметрия и доп-функции (GET, безопасно)

Перебрать чтением `operation=0xFA`, посмотреть осмысленность ответа:
- [ ] `cmd 0x0D` — обороты вентиляторов (CPU `[0..1]`, GPU `[2..3]`, SYS `[6..7]` LE).
- [ ] `cmd 0x16` — температура CPU (°C)? Подтвердить (коллизия с внутренним кодом).
- [ ] `cmd 0x13` — тип БП (1=USB-C, 2=DC).
- [ ] `cmd 0x12` — яркость подсветки клавиатуры (GET), затем аккуратно SET 0..10 / 128.
- [ ] `cmd 0x09` — режим GPU (GET). SET **не трогать** пока не поймём последствия (риск потерять вывод).

## Вопрос 6. Общая безопасность перебора

- [ ] Перебор неизвестных `cmd` делать **только с `operation=0xFA` (GET)**.
- [ ] SET на неизвестные функции не слать — можно записать в прошивку что-то нежелательное.
- [ ] Вести журнал: `cmd`, отправленный буфер, `OutData` — сложить в `../reference/probe-log.md`.

## Итог: что разблокирует эти ответы

| Ответ | Решение в архитектуре |
|-------|------------------------|
| В1 admin хватает + В3 события из сессии ловятся | **План А** — только трей-резидент, без службы |
| В1 нужен SYSTEM, или В3 события не ловятся | **План Б** — минимальная служба |
| В2 защита слетает после сна | Обязателен **ChargeGuard** |
| В4 модель `TM24*` | Показывать режим **Auto (0x09)** |
