# Что ещё полезного есть в MIControl (research)

Итог разбора всего проекта `reserch/micontrol`. Цель — что реально стоит перенести к нам.

## Главный вывод
MIControl через WMI дёргает **только 3 функции**: режим `0x08`, микрофон `0x0A`, заряд `0x10` — все уже реализованы у нас. Вся остальная «фишковость» = **обработка событий Fn-клавиш** (`HID_EVENT20`) + обычные действия Windows. Никаких скрытых WMI-функций нет.

Многие из этих клавиш на нашем ноуте, вероятно, **не работают без вендорского софта** — вот где основная ценность.

## Полная карта событий клавиш (`OnSpecialButtonPressed`)
`EventDetail[1]`=cmd, `[2]`=arg. Коды **зависят от модели** (у MIControl Mi=0x18/0x19, у нас 0x25/0x26 — проверять на своём железе).

| cmd | Клавиша | Что делает MIControl |
|-----|---------|----------------------|
| `0x16` | Смена режима (arg 1/2/3/4/9 = Баланс/Тихий/Турбо/Полный/Авто) | обновить UI + splash |
| `0x21` | Микрофон (0/1) | синхр. с системным mute + splash |
| `0x05` | Подсветка клавиатуры (0–10, `0x80`=авто) | splash с уровнем |
| `0x07` | Fn-Lock (0/1) | splash |
| `0x09` | Caps Lock (0/1) | splash |
| `0x1B` | Клавиша «Настройки» | открыть `ms-settings:` |
| `0x01` (arg 0) | Клавиша «Проекция» | эмулировать Win+P |
| `0x01` (arg 2) | Неправильный/слабый зарядник | предупреждение |
| `0x02` | Клавиша «Скриншот» | запустить `snippingtool.exe` |
| `0x13`/`0x1A` | Клавиша частоты экрана | циклически менять Гц (`ChangeDisplaySettings`) |
| `0x10` | Клавиша тачпада | вкл/выкл тачпад (реестр + `EnableTouchPad`) |
| `0xA0` | Камера/приватность (0/1) | splash |
| `0x18`/`0x19` | Mi-кнопка | пользовательское действие / показать окно |

## Что перенести к нам — по приоритету

### Tier 1 — высокая ценность, инфраструктура уже есть (слушатель событий)
1. **Синхронизация режима с физической клавишей** (`0x16`): когда режим меняют клавишей, обновлять наш динамический значок трея + показать OSD. Без этого значок «отстаёт» при переключении с клавиатуры.
2. **Оживить «мёртвые» клавиши**, которые шлют событие, но без софта ничего не делают:
   - «Настройки» (`0x1B`) → открыть Параметры Windows.
   - «Проекция» (`0x01`) → Win+P.
   - «Скриншот» (`0x02`, если есть) → «Ножницы» (`ms-screenclip:` / `Win+Shift+S`).
   - Клавиша частоты экрана → циклически менять Гц.
   - Клавиша тачпада → вкл/выкл тачпад.
   Это реально возвращает функциональность клавиатуры.
3. **Микрофон** (`0x21`): по нажатию синхронизировать системный mute (Core Audio `IAudioEndpointVolume`) + OSD.

### Tier 2 — приятная обратная связь (OSD)
- Подсветка клавиатуры (`0x05`) → OSD с уровнем (SET через WMI на нашей модели недоступен, только индикация).
- Fn-Lock (`0x07`), Камера (`0xA0`) → OSD. Caps Lock — можно пропустить (Windows и так показывает).
- Предупреждение «неправильный зарядник» (`0x01`/arg2) → OSD/уведомление.

### Tier 3 — обычные функции Windows (больше кода, к прошивке не относятся)
- Переключатель частоты экрана в меню/панели (`ChangeDisplaySettings`).
- Тачпад вкл/выкл в меню.
- Экран «О системе» (`Win32_BIOS`, `Win32_ComputerSystem` — серийник, версия BIOS, модель).

### Не берём
- ICC-профили, splash-скины, служба+pipe+protobuf (их архитектура), GPU-режим/вентиляторы/температура (на нашей модели WMI не отдаёт).

## Как вышло (актуализация 2026-07-10)
- Карта кодов снята → [07-keymap.md](07-keymap.md); весь Tier 1 и Fn-Lock из Tier 2 реализованы (v0.3–v0.5.3).
- Из Tier 2 остался только OSD «слабый зарядник» (`0x01`/value 2) — в планах.
- WMI `0x0C` — исследована и закрыта: рудиментный слот TPLock игровой таблицы MIFS,
  см. [01-wmi-protocol.md](01-wmi-protocol.md).
