# ТЗ на иконки (SVG) — Xi Control

Набор монохромных line-иконок в едином стиле. Я потом рендерю их в приложении
(перекраска под контекст + любой размер/DPI), поэтому важны правила ниже.

## Формат и стиль (обязательно)

- **SVG**, по одному файлу на иконку.
- **`viewBox="0 0 24 24"`** у всех (единая сетка). Рисунок центрирован, «живая» область ~20×20 (поля ~2).
- **Стиль — line/контур** (как Lucide / Feather / Tabler): без заливок-силуэтов.
  - `fill="none"`, рисуем штрихом.
  - `stroke-width="2"` (единая толщина у всех; для мелких/сложных допускаю 1.75–2.25, но держим ровно).
  - `stroke-linecap="round"`, `stroke-linejoin="round"`.
- **Цвет — только `currentColor`.** Все `stroke="currentColor"` (и `fill="currentColor"`, если где-то нужна заливка-акцент).
  🔴 **Никаких зашитых цветов** (`#000`, `black`, конкретные hex) — иначе я не смогу перекрашивать под тему/режим.
- Один цвет на иконку (монохром). Без градиентов, фильтров, теней, растровых вставок, текста, `<use>`/внешних ссылок.
- Только `<path>` и базовые примитивы (`<circle>`, `<rect rx>`, `<line>`, `<polyline>`). Пути — абсолютными координатами предпочтительно.
- Оптический баланс: все иконки должны выглядеть **одного визуального размера и веса** рядом друг с другом.

## Пример «рыбы» (структура файла)

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24"
     fill="none" stroke="currentColor" stroke-width="2"
     stroke-linecap="round" stroke-linejoin="round">
  <!-- пути иконки -->
</svg>
```

## Доставка
- Папка: `xi_control/assets/svg/`, имена файлов — как в таблицах.
- Все в одном стиле/сетке/толщине.

## Как я это подключу (для контекста)
- Рендерю SVG в приложении (библиотека Svg.NET или свой парсер путей) в `Bitmap` нужного размера/DPI, с перекраской через `currentColor` (белый/тёмный под тему трея, зелёный/синий/оранжевый под режимы, акцент для активного).
- Кэширую по (иконка × размер × цвет). Заменю текущий процедурный `Icons.cs`.
- Поэтому **`currentColor` — самое важное требование**: он даёт мне перекраску одной строкой.
